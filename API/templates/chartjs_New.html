FAQ

Optical character recognition, often abbreviated as OCR, is a tool to convert physical text into machine encoded and recognizable text. As part of the encoded character recognition process, OCR works commonly alongside OMR (optimal mark recognition), BCR (bar code readers), QR (quick response) scanners, HTR (handwritten text recognition) and MICR (magnetic ink character readers).

Physical text is often sourced from checks, labels, documents, invoices, bills, claim forms etc. and can be faxed, printed, e-stored, photographed, handwritten or scanned images.

 Most images in the healthcare space are a combination of these with unwanted visual noise emanating from ink smears, paper folds, rubber stamps, water marks, over-writes,  logos, graphics, signatures, illustrations etc.

 HGS Digital’s in-house and custom developed OCR stands out from other COTS solutions by offering a combination of recommended and mandatory pre-processing engines. Geometric corrections allow for image repositioning and fix scale, orientation and alignment issues. Radiometric corrections allow for image-centric pixel-level corrections. Additionally, the solutions accommodate for all legacy hardware thereby mitigating costs of upgrades.

These work in tandem to make any quality image more receptible to the OCR engine thus enhancing the quality of the extracted  output.

HGS Digital developed geometric image pre-processing involves use of various techniques from image transformations, feature detection-mapping-description-definition-matching to image registrations and homography.

Numerous algorithms stretching from hashing, SIFT-SURF,  RANSAC, FLANN, Brute Force, ORBH, Edge-Contour-Boundary detections, Hough and Affine transformations achieve these purposes.

HGS Digital developed radiometric image pre-processing involves a 2-stage omni-level image quality augmentation approach using multiple techniques from image segmentation, cleaning and super resolutions. These are achieved through the selective usage of:

  • Gray scale conversion
  • Otsu’s thresholding and binarization
  • Filtering
  • Bounding box layout analysis
  • Thinning
  • Edge detection
  • Dilation, Erosion & Filling
  • Image morphology

While super resolution is seen via:

  • Contrast adjustment
  • Rescaling and resizing via interpolation (Nearest Neighbor, Bilinear, Cubic, Bicubic and Spline)
  • SR-GAN
  • N-in-N Deep CNN

HGS Digital developed python-based OCR engine which then consumes the pre-processed output is a custom built solution incorporating in-house nodes working side-by-side with open source solutions from OpenCV, TensorFlow and Keras.

HGS Digital developed python-based QR, MICR, BCR, OMR and HTR tools work with the same libraries and custom models while incorporating in-house pre-processing from (in addition to above):

  • Document layout analysis
    • Geometric Layout analysis
    • Logical layout analysis
  • Edge-Contour Detection
  • Template matching

The UI developed for the same involves a simple file selection tool that allows for any type of image selection (jpeg, tiff, gif & pdf), display of raw and pre-processed images, extraction of image metadata for efficiency metric compilation and display of extracted text in a downloadable, searchable and on-screen editable format.

HGS Digital designed basic (default) and advanced NLP based post-processing modules allow for automatic and manual changes to be made to the final output before the download as csv/xlsx/txt/pdf/doc option is enforced.

The post-processing module borrows several features from:

  • Text pre-processing (ex: whitespace removals)
  • Text processing (ex: date field definitions)
  • Fuzzy Matching
  • BM25
  • Cosine Similarity
  • TF-IDF
  • Named entity recognition

Apart from select techniques from morphological, semantic, syntactic and lexical analysis.

 It is recommended to deploy images that have a print resolution of at least 80 dpi, image resolution at a minimum of 0.38 mp, font size ≥ 8 points stored @ 11 pixels and in English language only.

  • Tabular output
  • Banner output
  • OMR output

  • HTR output
